    链接绑定

    在链接绑定中，Guice 将类型映射到它的实现。即使用链接绑定 bind(SpellChecker.class).to(SpellCheckerImpl.class); 将 SpellChecker 接口与其实现 SpellCheckImpl 进行了映射。

    public class GuiceTester2 {
    public static void main(String[] args) {
        Injector injector = Guice.createInjector(new TestEditorModule2());
        TestEditor2 instance = injector.getInstance(TestEditor2.class);
        System.out.println(instance.hashCode());

        TestEditor2 instance1 = injector.getInstance(TestEditor2.class);
        System.out.println(instance1.hashCode());

    }
}

@Singleton
class TestEditor2{
    private SpellChecker2 spellChecker2;

    @Inject
    public void setSpellChecker2(SpellChecker2 spellChecker2){
        this.spellChecker2 = spellChecker2;
    }
    public TestEditor2(){ }

    public void makeSpellCheck(){
        spellChecker2.checkSpelling();
    }

    public double getSpellCheckerId(){
        return spellChecker2.getId();
    }
}

//Bind Module
class TestEditorModule2 extends AbstractModule{

    @Override
    protected void configure() {
        bind(SpellChecker2.class).to(SpellCheckerImpl2.class);
    }
}


interface SpellChecker2{
    public double getId();
    public void checkSpelling();
}

@Singleton
class SpellCheckerImpl2 implements SpellChecker2{

    double id;
    public SpellCheckerImpl2(){
        id = Math.random();
    }

    @Override
    public double getId() {
        return id;
    }

    @Override
    public void checkSpelling() {
        System.out.println("Inside checkSpelling.");
    }
}

    @Named 注解注入的方式，能够注解多个子类或者实现类，可以通过这种方式调用不同子类或实现类的方法。

    自定义注解

    使用@Named注释会存在一些风险，比如使用了一个未经信任的名字作为绑定的名字，则可能会导致安全漏洞。例如，如果名字是用户提供的，则可能会导致攻击者注入恶意代码。

    因此，尽管@Named注释可以使代码更加可读，但是在某些情况下使用它可能不是最安全的选择。

    使用自定义注解的方式，仿照@Named的形式实现自定义注解可以解决这一缺点

    1.首先是自定义注解

    @Qualifier
@Target({FIELD,PARAMETER,METHOD})
@Retention(RUNTIME)
public @interface AssertProvider {
    Provider value();
}

    2.自定义注解中Provider的值使用枚举写入

    public enum Provider {
    TestNG,
    JUnit,
    Other
}

    3.实现AssertProvier，AssertProviderImp中重写接口方法，并添加重写hashCode方法通过验证。

    public class AssertProviderImp implements AssertProvider{
    private Provider value;
    public AssertProviderImp(Provider value){
        this.value = value;
    }

    @Override
    public Provider value() {
        return this.value;
    }

    @Override
    public int hashCode() {
        //This is specified in java.lang.Annotation.
        //(name.hashCode() * 127) ^ value.hashCode()
        return (127 * "value".hashCode()) ^ value.hashCode();
    }

    @Override
    public boolean equals(Object obj) {
        if (!(obj instanceof AssertProvider)){
            return false;
        }
        AssertProvider other = (AssertProvider) obj;
        return value().equals(other.value());
    }

    @Override
    public String toString() {
        return "@" + AssertProvider.class.getName() + "(" + Annotations.memberValueString("value", value) + ")";
    }

    @Override
    public Class<? extends Annotation> annotationType() {
        return AssertProvider.class;
    }
}

    4.编写AssertProviders，仿照@Named的方式，形成工厂类，通过AssertProviders调用静态方法，获取对象。

    public class AssertProviders {
    public static AssertProvider assertProvider(Provider value){
        return new AssertProviderImp(value);
    }
}

    5.在AbstactModule中的绑定方式为:

            bind(Asserter.class).annotatedWith(AssertProviders.assertProvider(JUnit)).to(JunitAsserter.class);
        bind(Asserter.class).annotatedWith(AssertProviders.assertProvider(TestNG)).to(TestNGAsserter.class);

    6.实现方法

    public class TextEditor {
    private SpellChecker spellChecker;
    private Asserter asserter;

    @Inject
    public TextEditor(
            SpellChecker spellChecker,
            @AssertProvider(JUnit) Asserter asserter
            )
    {
        this.spellChecker = spellChecker;
        this.asserter = asserter;
    }

    public void makeCheckSpelling(){
        spellChecker.checkSpelling();
        asserter.equal(1,1);
    }
}

    即时绑定

    由于绑定是在绑定模块中定义的，Guice 会在需要注入依赖项时使用它们。如果绑定不存在，它可以尝试创建即时绑定。绑定模块中存在的绑定称为显示绑定并且具有更高的优先级，而即时绑定为隐式绑定。如果两种类型的绑定都存在，则考虑显示绑定进行映射。

    绑定类型 描述
    Injectable
    Constructor    		 非私有、无参数的构造函数有资格进行即时绑定。另一种方法是使用 @Inject 注释来注释构造函数。
    @ImplementedBy
    annotation    		 @ImplementedBy 注释告诉指南有关实现类的信息。在这种情况下，绑定模块中不需要绑定。
    @ProvidedBy
    annotation    		 @ProvidedBy 注解告诉指南关于实现类的提供者。在这种情况下，绑定模块中不需要绑定。

    injectMembers方法是Guice的一个方法，它可以将一个对象中的所有带@Inject注解的属性或方法注入相应的依赖。

    例如，如果你有一个类Foo，它有一个带@Inject注解的属性bar，那么你可以使用injectMembers方法来将bar属性注入相应的依赖，如下所示：

    // 假设Foo类有一个名为bar的带@Inject注解的属性
Foo foo = new Foo();

// 创建Guice的Injector对象
Injector injector = Guice.createInjector(...);

// 使用injectMembers方法将foo对象中所有带@Inject注解的属性注入相应的依赖
injector.injectMembers(foo);

    总之，injectMembers方法可以让你在不用手动实例化依赖的情况下，自动将依赖注入到一个对象中，大大简化了依赖注入的工作。

    杂项示例

    作用域

    Guice 每次提供值作为其默认行为时都会返回一个新实例。它可以通过范围进行配置。以下是 Guice 支持的范围：

    • @Singleton - 应用程序生存期的单个实例。@Singleton对象必须是线程安全的。
    • @SessionScoped - Web 应用程序特定会话的单个实例。@SessionScoped对象必须是线程安全的。
    • @RequestScoped - Web 应用程序的特定请求的单个实例。@RequestScoped对象不需要是线程安全的。

    应用范围的方法

    AOP

    Guice 是一种依赖注入框架，它可以帮助开发人员实现依赖注入。AOP 是面向切面编程的缩写，它是一种编程技术，可以在不修改源代码的情况下提供应用程序的某些功能。在 Guice 中，AOP 可以用来实现横切关注点，这些关注点是应用程序的特定功能，比如日志记录、事务管理等。

    重要类

    • 匹配器 - 匹配器是接受或拒绝值的接口。在 Guice AOP 中，我们需要两个匹配器：一个用于定义哪些类参与，另一个用于这些类的方法。
    • 方法拦截器 - 方法拦截器在调用匹配方法时执行。他们可以检查调用：方法、其参数和接收实例。我们可以执行横切逻辑，然后委托给底层方法。最后，我们可以检查返回值或异常并返回。
    public class GuiceTester {
    public static void main(String[] args) {
        Injector injector = Guice.createInjector(new TestEditorModule());
        TestEditor instance = injector.getInstance(TestEditor.class);
        instance.makeCheckSpelling();
    }
}

//Binding Module
class TestEditorModule extends AbstractModule{
    @Override
    protected void configure() {
        bind(SpellChecker.class).to(SpellCheckerImpl.class);
        bindInterceptor(Matchers.any(),
                Matchers.annotatedWith(CallTracker.class),
                new CallTrackerService());
    }
}

//interface
interface SpellChecker{
    public void checkSpelling();
}

//implement
class SpellCheckerImpl implements SpellChecker{

    @Override @CallTracker
    public void checkSpelling() {
        System.out.println("Inside checkSpelling.");
    }
}

@Retention(RetentionPolicy.RUNTIME) @Target(ElementType.METHOD)
@interface CallTracker{}

class CallTrackerService implements MethodInterceptor{
    @Override
    public Object invoke(MethodInvocation methodInvocation) throws Throwable {
        System.out.println("Before " + methodInvocation.getMethod().getName());
        Object result = methodInvocation.proceed();
        System.out.println("After " + methodInvocation.getMethod().getName());
        return result;
    }
}

    @CallTracker 是 Guice 中的一个注释，它可以用来标注一个方法，表明该方法需要被跟踪。在实际应用中，通常需要定义一个拦截器来拦截标注了 @CallTracker 的方法，并实现跟踪逻辑。这样，就可以跟踪标注了 @CallTracker 的方法的调用情况，比如方法被调用的次数、调用时间等.
